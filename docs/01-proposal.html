1 プロジェクト概要

サービス名

香水メーカー（仮名称：c-corp）ワークショップ関連ページ改修。

概要

ワークショップ申込数の伸び悩みを課題とする香水メーカーからのWebサイト改修相談。スピード感を求められているため、2フェーズにわけて実施するものとする。

Phase 1：従来の2カラム構成は維持しつつ、分散していた情報を整理。ブランド既製品との相乗効果（シャワー効果）を意図し、トーンを高級路線へ再設計。
Phase 2：外部システム依存となっていた予約システムの構築。（既存メルマガ登録用システムに機能追加する想定）

ターゲットユーザー

香水ワークショップに興味関心を持つ層。

2 背景と課題

市場の現状

2025年から2034年にかけて、市場は年平均成長率（CAGR）7.40%で成長すると予測されている。また、「自分らしさ」を重視するZ世代により、パーソナライズ香水の需要が拡大していると考えられる。（参照：H&Iグローバルリサーチ株式会社2025年12月3日「香水の日本市場：香水、オーデコロン（EDC）、オードパルファム（EDP）、オーデトワレ（EDT）、その他2025年～2034年」 ）

一方で、他国とは異なり、日本では歴史的に「無臭であること」が清潔・礼儀と結びついている点に注意が必要であると考えられる。また、現代においては「スメハラ」「香害」という言葉に代表されるように「強い香り」に忌避感を抱く場合がある。

解決すべき課題

クライアントの認識している課題：
ワークショップ申込数の低迷

現行Webサイトを拝見しての課題：
ワークショップ導線の弱さ
イメージ醸成や第三者評価などのPUSH要素不足
情報が整理されておらず、理解・比較がしづらい状態
既製品とワークショップ間のトンマナ不一致
外部予約サイト依存

Webアプリケーション（サーバーサイド）の範囲 必須

ReservationApplication.javaを起点とする Spring Boot サーバーアプリケーションとして、以下の機能を提供する。
本アプリは、ワークショップ予約受付とメール配信登録を行う公開API、および運営向け管理APIで構成される。

  1. 予約機能（公開API）
    • 空き状況照会（コース・期間指定で日別/時間帯別の空き枠を返却）
    • 予約登録（日時・コース・人数・連絡先を受け取り予約番号を発番）
    • 予約時に、対象スロットの定員と既存予約人数を照合して満席判定を実施
  2. メール会員機能（公開API）
    • メールアドレスの登録
    • メールアドレスの配信停止（退会）
    • 登録/停止状態の更新管理
  3. 管理機能（管理API）
    • 直近予約一覧の取得
    • 予約集計（週次の時間帯別件数、月次の日別受付件数）
    • メール会員一覧の取得
    • メール会員集計（月次の日別登録数・解除数）
    • ログイン中管理者情報の取得
  4. 認証・運用基盤
    • 管理画面・管理APIは認証必須（Spring Security）
    • 公開APIは外部から利用可能（CORS設定あり）
    • ルートアクセス時はトップページへリダイレクト
  5. データ管理対象
    • 予約情報（予約番号、コース、日時、人数、連絡先）
    • メール会員情報（メールアドレス、有効/無効、更新日時）
    • スロット定員情報（日時ごとの定員設定）
