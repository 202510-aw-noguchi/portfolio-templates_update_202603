1 テスト計画

テスト対象・範囲 必須
テストの対象となる機能・画面と、テストの種類

機能一覧（仕様機能の整理）

1. コーポレートサイト表示
index / about / brands / press / magazine

2. ワークショップ案内表示
workshop.html、workshop/casual.html、workshop/holiday.html

3. ワークショップ予約
workshop/registration.html

- コース選択
- 予約可能日表示（空き状況API連携）
- 時間枠選択
- 参加人数・連絡先入力
- 予約確定（予約番号表示）

4. メールマガジン
mail.html

- 登録
- 配信停止

5. 管理者ログイン
/login.html（Spring Security）

6. 管理画面（Backoffice）
/admin.html

- 予約一覧
- 予約サマリ（週次・月次）
- メルマガ一覧
- メルマガサマリ（登録/停止推移）
- ログアウト

7. API

- 予約: GET /api/reservations/availability, POST /api/reservations
- メルマガ: POST /api/mail/subscriptions/register, POST /api/mail/subscriptions/unsubscribe
- 管理: GET /api/admin/*

テスト対象の範囲

- 対象（In Scope）

- フロント画面の表示・入力・遷移
- 予約/メルマガのAPI連携
- 管理者認証と管理画面のデータ表示
- 入力バリデーション（必須、形式、上限）

- 非対象（Out of Scope）

- 本番インフラ監視、外部メール配信基盤の実配信品質
- 大規模負荷試験（必要なら別計画で実施）

テスト種別

1. 機能テスト

- 正常系: 予約作成、空き枠取得、メルマガ登録/停止、管理画面取得
- 異常系: 必須不足、不正メール形式、満席枠予約、未ログイン時の管理APIアクセス

2. 画面テスト

- 主要画面の表示崩れ、レスポンシブ、リンク遷移
- フォーム操作性（入力制限、エラー表示、モーダル表示）
- 管理画面タブ切替・再読み込み・ログアウト導線

3. セキュリティテスト

- 認証/認可: /admin.html と /api/admin/* のアクセス制御
- 入力値対策: XSS（表示エスケープ）、SQLインジェクション相当の不正入力
- CSRF/CORS/セッション周りの基本確認（ログアウト後アクセス不可など）

テスト環境

- OS

- Windows 11
- macOS 14以上

- ブラウザ

- Google Chrome 最新版
- Microsoft Edge 最新版
- Safari 最新版（macOS）
- 任意追加: Firefox 最新版（互換性確認）

- バックエンド

- Java 17
- Spring Boot（reservation-app）
- H2 Database
