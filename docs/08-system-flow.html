データの流れ
Browser -> Controller(API) -> Service -> Repository -> DB
Browser <- JSON Response <- Controller <- Service <- Repository <- DB

画面遷移の流れ
Browser -> /login.html, /admin.html (static)
admin.html -> /api/admin/* を fetch
registration.html -> /api/reservations/* を fetch
mail.html -> /api/mail/subscriptions/* を fetch

レイヤー構成の説明
各レイヤーの役割を表形式で説明
レイヤー 役割 主なクラス
クライアント（ブラウザ） 画面表示、ユーザー入力、API呼び出し（fetch）を担当 workshop/registration.html, mail.html, reservation-app/src/main/resources/static/admin.html, reservation-app/src/main/resources/static/login.html
Controller層 HTTPリクエスト受付、入力受け取り、Service呼び出し、レスポンス返却 ReservationController, MailSubscriptionController, AdminController, CsrfTokenController, RootRedirectController
Service層 業務ロジック（予約可否判定、集計、メルマガ登録/解除など）を実装 ReservationService, MailSubscriptionService
Repository層 DBアクセスを抽象化し、エンティティの永続化・検索を担当 ReservationRepository, MailSubscriptionRepository, SlotCapacityRepository
データベース層 永続データの保存・参照 H2（reservationdb）、エンティティ: Reservation, MailSubscription, SlotCapacity
View層（Thymeleaf） サーバーサイドHTMLテンプレート生成 なし（本アプリはThymeleaf未使用、静的HTML配信）

4 データフロー

予約登録 (workshop/registration.html -> /api/reservations -> reservation)
[Screen: workshop/registration.html]
  input fields:
    course(String), reserveDate(String "yyyy-MM-dd"), reserveTime(String),
    participantCount(String input -> JS Number), contactName(String),
    contactPhone(String), contactEmail(String)

  JS manual payload mapping:
    payload = {
      course: getSelectedCourse(),                         // String
      reserveDate: dateInput.value,                        // String
      reserveTime: selectedTime.value,                     // String
      participantCount: Number(participant-count.value),   // String -> Number
      contactName: contact-name.value,                     // String
      contactPhone: contact-phone.value,                   // String
      contactEmail: contact-email.value                    // String
    }

  POST /api/reservations (application/json)
        |
        v
[ReservationController#createReservation]
  @Valid @RequestBody ReservationRequest request
  ReservationRequest fields:
    course:String
    reserveDate:LocalDate
    reserveTime:String
    participantCount:int
    contactName:String
    contactPhone:String
    contactEmail:String

  type conversion (Jackson):
    reserveDate: String("yyyy-MM-dd") -> LocalDate
    participantCount: JSON Number -> int
        |
        v
[ReservationService#createReservation]
  conversion/mapping method: 手動マッピング (setterを1件ずつ)
  - normalizeCourse(course.trim().toLowerCase())
  - new Reservation()
      setReservationNumber("TMP-...")                 // String生成
      setCourse(request.course normalized)            // String
      setReserveDate(request.reserveDate)             // LocalDate
      setReserveTime(request.reserveTime)             // String
      setParticipantCount(request.participantCount)   // int
      setContactName(request.contactName)             // String
      setContactPhone(request.contactPhone)           // String
      setContactEmail(request.contactEmail)           // String

  save #1:
    reservationRepository.save(reservation)
    (JPA/Hibernate -> INSERT into reservation)
  then:
    reservationNumber = "RSV-" + reserveDate.format("yyyyMMdd") + "-" + id(5桁)
    // LocalDate -> String(yyyyMMdd)
    saved.setReservationNumber(reservationNumber)
  save #2:
    reservationRepository.save(saved)
    (JPA/Hibernate -> UPDATE reservation)
        |
        v
[DB table: reservation]
  columns:
    id(BIGINT, PK, IDENTITY)
    reservation_number(VARCHAR)
    course(VARCHAR)
    reserve_date(DATE)                   // LocalDate
    reserve_time(VARCHAR)
    participant_count(INT)
    contact_name(VARCHAR)
    contact_phone(VARCHAR)
    contact_email(VARCHAR)
    created_at(TIMESTAMP)                // LocalDateTime (@PrePersistで自動設定)

  保存方法:
    Spring Data JPA save()
メルマガ登録 (mail.html -> /api/mail/subscriptions/register -> table save) 
[Screen: mail.html]
  input fields:
    email(String)

  JS manual payload mapping:
    { email: registerForm.email.value }   // String
  POST /api/mail/subscriptions/register
        |
        v
[MailSubscriptionController#register]
  @Valid @RequestBody MailSubscriptionRequest request
  MailSubscriptionRequest fields:
    email:String
        |
        v
[MailSubscriptionService#register]
  conversion:
    normalizeEmail(email.trim().toLowerCase())    // String正規化
  mapping:
    (A) not found:
      new MailSubscription()
        setEmail(email)
        setActive(true)
        setUnsubscribedAt(null)
      mailSubscriptionRepository.save(created)     // INSERT
    (B) already active:
      no DB write
    (C) exists inactive:
      setActive(true)
      setUnsubscribedAt(null)
      save(subscription)                           // UPDATE
        |
        v
[DB table: mail_subscription]
  columns:
    id(BIGINT, PK, IDENTITY)
    email(VARCHAR, UNIQUE)
    active(BOOLEAN)
    created_at(TIMESTAMP)              // @PrePersist
    updated_at(TIMESTAMP)              // @PrePersist/@PreUpdate
    unsubscribed_at(TIMESTAMP, NULL)

  保存方法:
    Spring Data JPA save()
メルマガ解除 (mail.html -> /api/mail/subscriptions/unsubscribe -> table update)
[Screen: mail.html]
  input fields:
    unsubscribe_email(String)

  JS manual payload mapping:
    { email: unsubscribeForm.unsubscribe_email.value }  // String
  POST /api/mail/subscriptions/unsubscribe
        |
        v
[MailSubscriptionController#unsubscribe]
  @Valid @RequestBody MailSubscriptionRequest
        |
        v
[MailSubscriptionService#unsubscribe]
  conversion:
    normalizeEmail(email.trim().toLowerCase())
  mapping/update:
    (A) not found -> no DB write
    (B) already unsubscribed -> no DB write
    (C) active:
      setActive(false)
      setUnsubscribedAt(LocalDateTime.now())    // now()で型生成
      save(subscription)                         // UPDATE
        |
        v
[DB table: mail_subscription]
  updated columns:
    active: true -> false
    unsubscribed_at: null -> timestamp
    updated_at: auto-updated by @PreUpdate

      |
      v
+-------------------------+

+-------------------------+
      |
      v

+--------------------------------------------------+
      |
      v
